ATTRIBUTE BREAK

This functionality has two parts: Setting an AlternativeText and supplying a label to get the data for parts.

END CONNECTION

These are special labels that are evaluated for end connections of an isometric.

1.       Edit Options for the ShawIso_D style

2.       Select Labels.EndConnection to select an end label

4.       For the END-CONNECTION-PIPELINE label, the piping part connected to the last object on the current iso line is passed to the label. It is the label which then retrieves the appropriate continuation item based on the kind of isometric being run

LAB-12: Change Management

 

Change Management on a piping isometric drawing refers to the fact that the same information needs to appear on the same sheet of an isometric drawing each time you extract the drawing.

Objective: To implement change management to facilitate control of single-sheet isos

1.       Edit Options for the style ShawIso_C created in Lab 3 above

2.       Set Drawing.Content.ChangeManagementEnabled option to True

3.       Set the Drawing.Controls.PipeLineSplitting option to 90 (%)

CAE Notes:

We have found that currently this switch is being ignored when Drawing.Content.ChangeManagementEnabled is set to True.

In order to take advantage of this feature in production, this value can be set to 110% when change management is set to False to decrease the likelihood of an iso being extracted on more than one sheet.

4.       To handle material list overflows from sheet to sheet without breaking change management, use isometric option settings:

a.       MaterialList.MaterialListOverflow to Continuation Sheet

b.      MaterialList.OverflowDrawingID to Alpha suffix (e.g: 2A, 3A, etc).

c.       Set MaterialList.Drw1of1OnSingleIsos to True

5.       With all these things done, part items will remain on the same sheet if new revs are issued. However, perhaps due to a bug in the system, currently change management forces all items into a single sheet at almost any cost

6.       Extract any pipeline, i.e. line 1001-P, and review sheet content

7.       Switch to Piping task and model some new branches with components on line 1001-P. For this example several branches were added to push the limits (not required for student to do):

8.       Extract line 1001-P again and review the sheets contents

9.       Notice how Isogen attempts to keep all items in one sheet, with overflow sheets for the MTO when necessary (sheets 2 and 3 in the illustration)

10.   Edit Options for the style ShawIso_C

11.   Set Drawing.Content.ChangeManagementEnabled option to False

12.   Extract line 1001-P again and review the contents

13.   A basic multi-sheet drawing is generated

14.   Return to the Piping task, set the locate tool to piperun, and delete the extra piperuns created for this lab

LAB-14: Iso Control Content

 

The Drawings.Content.HonorSpoolAtBranch option allows you to drive the content for isometric drawings for pipelines according to spools.

Objective: To manipulate spooling to force stub-ins to be a part of the headers iso

1.       In Piping task, create two new pipeline systems in unit U01: LINE_01 and LINE_02

2.       Using piping spec 1C0031 model in space a 10in header pipeline system under LINE_01, then a 6in RWeld (default) branch as LINE_02 with a flange take down joint 12in from the header CL as shown

3.       Use the spools command to generate the spools with the Include Stub-Ins with Header Spool property set to True

4.       Switch to Drawings and Reports task and edit Options for the ShawIso_D style in Unit01, expand the tree to Drawing.Content.HonorSpoolAtBranch and set to True

5.       Switch to Drawings and Reports task and create iso drawing for LINE_01. It should include stub-in and flange despite their belonging to line system LINE_02

LAB-15: Iso Weld Numbers Control

 

Drawing.Welds.WeldNumberLabel Specifies a label to use with the weld number on the isometric drawing. There are two delivered weld number labels you can use with this option: Piping Isometric Weld Sequence Number and Piping isometric Weld Type and Sequence Number.

Objective: To activate and manipulate the display of welds on the iso

1.       In Piping task, use the Sequence Objects command to generate weld numbers for line 1002-P

2.       Switch to Drawings and Reports, Edit Options for style ShawIso_D

3.       Expand to Drawing.Welds.WeldNumberLabel

4.       Click the ellipsis button in the field to display the Catalog Labels dialog box and select label Piping Isometric Weld Seq Number

5.       Set the Drawing.Welds.ShowWeldNumbers option to True

6.       Save settings to catalog and exit

7.       FYI weld identifiers with spaces are not supported in the label

8.       Update iso of 1002-P and review the weld numbers on the drawing



Implementing a custom Shaw weld number label

9.       Edit Options for style ShawIso_D

10.   Expand to Drawing.Welds.WeldNumberLabel

11.   Click the ellipsis button in the field to display the Catalog Labels dialog box and select label ShawTrainingWeldID located under the Piping Isometric folder for catalog labels (contact CAE if label is missing)

12.   Verify that Drawing.Welds.ShowWeldNumbers option is set to True

13.   Save settings to catalog and exit

14.   Update iso of 1002-P and review the weld numbers on the drawing

15.   The weld numbers should display a different format that makes them unique across the whole plant. In this case the pipeline name is suffixed with the weld number to form the weld number
