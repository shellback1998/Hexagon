General concepts to remember:

1.       The parametric driving dimensions that the user can modify appear in black on the illustrations of the dimensional legends

2.       Hard-coded and calculated dimensions are shown in red

3.       Equipments building blocks are placed from the coordinate location of CP1 out. Therefore if the location of nozzles that default at the same location as CP1 needs to change, then the placement point for the whole equipment will need to be changed.  An example of this is the nozzles on an air cooler, which are always aligned with CP1

4.       Due to conflicts within the software, we had to de-activate the nozzles driving dimensions in the parametric forms.  This means that after the parametric equipment is placed, to revise a nozzle location the user must use the location tab in the nozzles own attributes form.  Nozzle attributes (NPD, endprep) can be revised in either interface.
NOTE: If the user revises a radial nozzles face location, and then the diameter of the equipment changes later at any time, the radial distance revised will be overridden and the nozzle will be adjusted out or in to maintain a min clearance of 300mm from nozzle end to eqp outer surface.  Likewise, an axial nozzle that had its face location revised will be automatically adjusted in or out when equipment length is revised, so as to maintain a min clearance of 300mm from the equipment surface.

5.       Parametric nozzles can be selected individually for revision or to be deleted. 

6.       New nozzles can be added manually to a parametric as well as other graphic components as needed to further enhance the look of the final equipment item

7.       Some parametric routines dont like 0-length values, which can result in loss of display of portions of a parametric. Whenever this happens replace the 0mm value with 0.0001mm. An example is the pier height for some of the parametrics.  Typically using a 0mm for pier height should only eliminate the pier from being displayed. However in some cases doing this also turns off display of other graphics.  Whenever this happens do as instructed above and replace the 0mm with 0.0001mm in the pier height field.

8.       SP3D parametrics use shared interfaces to optimize performance and database space.  This means that you will often see more parametric values available than what is used by the active equipment parametric form.  For example, the Shaw Air Cooler for FEED projects has a built in function that calculates the Fan Diameter, therefore the user does not need to enter this value. However it uses the same interface as other Air Coolers that may allow the Fan Diameter to be entered, for this reason you will see the Fan Diameter available as a key in field for the Shaw Air Cooler, because it is part of the interface shared by all air coolers. If you should enter a value for the FEED air cooler fan diameter however, the system will simply ignore that value and continue to calculate it.  Therefore, limit yourself to entering only the values that are requested in the parametrics dimensional legend.

 

 

AIR COOLER

1.       Select the Place Equipment command

2.       Navigate the eqp catalog hierarchy to the air cooler parametric under the Shaw>FEED folder

3.       Select the air cooler from the window to the right, there is only one at this time, then OK

4.       In the Equipment Properties form enter the equipment name and system hierarchy location as usual

5.       Select the dimensional legend icon to see the parametric sketch.
All driving dimensions have been given defaults. To review those values and possibly change some of them you must access the interfaces that contain these values.

6.       In the Category pull down select Equipment Dimension

7.       Notice in the dimensional legend diagram that the needed values are A, B, and D (Cooler length, width, height) and are filled with defaults.  All other values in the Dimension interface can be ignored. In general, values that are not used by a parametric appear blank in the interface. Revise the values for these driving dimensions if needed, then select Equipment Support from the pull down menu

8.       Notice in the dimensional legend that parameter C is for the Support Height.  Revise the existing default if necessary.
Note: Defaults were preset in metric units.  To see your units displayed as metric in your properties forms, remember to use Tools>Options from the menu, and then select the Units of Measure tab to be able to select your default units for this session

9.       Select OK on the properties form

10.   Use Pinpoint to place the Air Cooler.  Notice that at the moment this and other Shaw parametrics have only one placement point.

11.   TIP: to relocate the air coolers reference point, right click on it and select Reference Coordinate System

a.       Next select the common app Move command

b.      The coolers reference coord sys will go into dynamic mode, use pinpoint (F8) to change its elevation to be the same as the bottom of the cooler legs:

c.       Now click on the cooler to select it as a whole equipment again

d.      Select the Move command

e.      Notice how as you move your cursor the default placement location for the whole equipment is the new reference coordinate system location at bottom-of-legs level

f.        Yet another way to override the reference point as you move equipment is to select the Move From button during Move command.  The system will then prompt you for a point to move from, select another graphic on the cooler or even another unrelated graphic to use as the point to move from.

 

NOTE TO USERS

For the remaining Shaw FEED parametrics the instructions will be less detailed than the cooler.  We will highlight the location of the needed parameters in the interfaces used by each parametric, and mention any details particular of the parametric being reviewed to facilitate its placement and modification.

U TUBE EXCHANGER

In the programming environment of the symbol development we do not have access to the active nominal size of the nozzles as global variables, so a quick work around was to have the user manually enter T1s size in this field.  You do not need to calculate the conversion value from inches to meters, simply enter the nom size with its units, i.e. 8in, and press enter. The system will convert that key-in to the active units for you in the form.

HORIZONTAL PUMP

For this pump, the nozzle position parameters A, B, C, and D, combined with the size of the base plate, will drive the location and size of the impeller and shaft casings, as well as the driver. So that changing the base plates dimensions alone is not enough to create a proportionally accurate pump.

NOZZLE MANIPULATIONS

 

To modify the location of a default parametric nozzle we will use the Pipe Nozzle Properties form.

Occurrence attributes like nozzle size and end preparation can be revised in this nozzle interface, or through the Equipment Properties forms Connection tab.

 

Using the Location tab from the Nozzle Properties will give the user greater flexibility in manipulating a nozzles location as compared to manipulating it from the equipment form.

Example:  To move and rotate a parametric nozzle along the body of its parent equipment, set the selection tools locate filter to Pipe Nozzle and follow these steps:

1.       Click on the nozzle to change location for


2.       Open the Nozzle Properties form


3.       Select the Location tab on the form


4.       Notice that the default placement type is Offset Skew.  This is the default from the CAXPERTS symbol editor as it is the most flexible and can be made to look like most types of nozzles from one interface:


5.       The selected nozzle on the heat exchanger is a radial nozzle, so that its driving parameters are N1, N2, and OR1.  They can be modified in this Offset Skew interface, or in the Radial interface to simplify visualization. In this example well rotate the nozzle to the bottom of the exchanger and 3.5m from the datum point along the equipments X-axis:




6.       You can modify the center-to-face dimension N2 (exaggerated for clarity):


7.       However, if the diameter of the exchanger should change, the parametric will resume control of the dimension N2 for the nozzle and recomputed its value, overriding whatever value you might have entered.  This will make it possible for this and all other parametric nozzles to be automatically adjusted when the diameter of the parent equipment changes to maintain a minimum 300mm clearance:

Similarly, axial parametric nozzles will be adjusted automatically when the length of a parent equipment changes.
